# mkultraman-v4 Launch Plan

**Date:** 2026-04-02
**Branch:** main
**Repo:** tyrannosaurusjr/mkultraman-v4
**Stack:** Astro v6 (static), CSS custom properties, Netlify Forms, Vercel deploy

---

## Problem Statement

mkultraman.com currently runs on WordPress + Divi. It performs poorly (Lighthouse mobile ~0.67), doesn't convert (fragmented positioning across 3 different identities), and requires expensive WordPress hosting overhead. The new site must convert SME owners into inquiry form submissions for digital infrastructure management services.

**Core offer:** "Your tech stack is costing you money. I fix it."
**Primary CTA:** Fill out the inquiry form.
**Audience:** SME owners and operators in Japan.

---

## Current State (v4 prototype as built)

The Astro v4 prototype exists at `~/Desktop/tyrannojr-github/mkultraman-v4` with:
- 76 pages building cleanly (73 blog posts + homepage + blog index + guild)
- Dark design system: `#111111` bg, `#C41E3A` accent, `#F0EDE8` text, Fraunces + Geist
- Components: Nav, Hero (with Netlify form), Problems, Services, Insights, About, Footer
- Local Lighthouse mobile: 0.82 performance, 0.94 accessibility, 1.00 best-practices, 1.00 SEO
- Blog migrated from WordPress with full Divi content extraction (4,000–11,000 chars/post)

**What's broken / missing:**
1. `--space-10` undefined token used in `Footer.astro:86` (gap collapses to 0)
2. `--space-40` undefined token used in `Footer.astro:115` (mobile gap collapses)
3. `/privacy` and `/terms` — linked in footer, both 404
4. No OG/Twitter meta tags in `Base.astro` (social share broken)
5. No canonical URL in `Base.astro`
6. No favicon (`/favicon.svg` referenced but missing)
7. `astro.config.mjs` has no `site:` URL — breaks sitemap and canonical generation
8. No `vercel.json` — deploy config not locked
9. No sitemap or robots.txt
10. No `_redirects` file — old WordPress URLs won't pass SEO juice to new site
11. Style tokens loaded from `/public/styles/` (not `/src/styles/`) — wrong path in `Base.astro`
12. Nav links Services/About as anchor links (`#services`, `#about`) — fine for homepage, broken from blog posts

## Implementation Plan

### Phase 1: Launch Blockers (execute tonight, ~45 min CC)

**1A. Fix CSS token bugs**
- `Footer.astro:86` — change `--space-10` to `--space-8`
- `Footer.astro:115` — change `--space-40` to `--space-48`

**1B. Fix global CSS path**
- `Base.astro:30` references `/styles/global.css`
- Tokens and global CSS are in `src/styles/` (compiled output), not `public/styles/`
- Astro serves component styles via Vite — remove the manual link tag, import styles via layout instead
- Add tokens import to `global.css` via `@import './tokens.css'`

**1C. Add OG/Twitter/canonical meta to Base.astro**
- Add props: `ogImage`, `canonicalURL`
- Add `<meta property="og:*">` and `<meta name="twitter:*">` tags
- Default OG image: `/images/og-default.png` (create 1200x630 dark card)
- Add `<link rel="canonical">` using `Astro.url`

**1D. Set site URL in astro.config.mjs**
- `site: 'https://mkultraman.com'`
- Add `@astrojs/sitemap` integration

**1E. Create favicon.svg**
- Simple "M" lettermark in Fraunces style on dark bg

**1F. Create /privacy page**
- Minimal GDPR/privacy policy page
- Uses Base layout, reading column, links back home

**1G. Create /terms page**
- Minimal terms of service page
- Uses Base layout, reading column, links back home

**1H. Create vercel.json**
```json
{
  "framework": "astro",
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        { "key": "X-Content-Type-Options", "value": "nosniff" },
        { "key": "X-Frame-Options", "value": "DENY" },
        { "key": "Referrer-Policy", "value": "strict-origin-when-cross-origin" }
      ]
    }
  ]
}
```

**1I. Create robots.txt and sitemap**
- `public/robots.txt` — allow all, sitemap link
- Sitemap auto-generated by `@astrojs/sitemap`

**1J. Create _redirects for WordPress URL migration**
- Map old WP paths to new Astro paths (same slugs, same structure)
- Catch-all 301 redirects for any WP admin / feed / XML paths

### Phase 2: SEO + Trust Layer (~30 min CC)

**2A. Add proof strip to Hero**
- Three stats (already have): 40+ SMEs / 10yr Japan / 30% avg spend cut
- Move stats below form on mobile, alongside on desktop

**2D. Add per-post OG meta**
- Blog post template already has title + description

**2E. Add Plausible analytics**
- Privacy-first, GDPR-compliant (important for Japan market)

### Phase 3: Design Polish (~20 min CC)
